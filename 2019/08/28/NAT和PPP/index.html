一、NAT

1.1 NAT的作用：将私有地址转化为公有地址

  • 私有地址：
    A类：10.0.0.0 - 10.255.255.255
    B类：172.16.0.0 - 172.31.255.255
    C类：192.168.0.0 - 192.168.255.255

    1.2 NAT的三种类型：

静态NAT：一对一，应用于内部网络服务器等这些对外发布的平台。
动态NAT：多对多，将公网申请多个公网IP地址，然后做成地址池。内部网络转化到这个地址上网。（企业中很少申请动态NAT：浪费）
过载NAT（overload）：又称PAT(端口多路复用)多对一，将多个私有内部地址映射到一个公网出口地址。在只有一个公有IP地址的情况下，要让很多用户都能够连接到因特网，可使用PAT的网络地址转换。

1.3 NAT配置

1.3.1 静态NAT配置

1)配置内置端口in和外端口out
R1(config)#int s0/3/0
R1(config-if)#ip nat outside
R1(config)#int range f0/0.1,f0/0.2
R1(config-if-range)#ip nat inside
2)配置地址池
R1(config)#ip nat pool A 61.1.1.3 61.1.1.8 netmask 255.255.255.0
3)设置ACL
R1(config)#access-list 1 permit any
R1(config)#int s0/3/0
R1(config-if)#ip access-group 1 out
R1(config)#int range f0/0.1,f0/0.2
R1(config-if-range)#ip access-group 1 in
4)静态NAT配置：
R1(config)#ip nat inside source static 192.168.0.1 61.1.1.1

1.3.2 动态NAT配置

1)配置内置端口in和外端口out
R1(config)#int s0/3/0
R1(config-if)#ip nat outside
R1(config)#int range f0/0.1,f0/0.2
R1(config-if-range)#ip nat inside
2)配置地址池
R1(config)#ip nat pool A 61.1.1.3 61.1.1.8 netmask 255.255.255.0
3)设置ACL
R1(config)#access-list 1 permit any
R1(config)#int s0/3/0
R1(config-if)#ip access-group 1 out
R1(config)#int range f0/0.1,f0/0.2
R1(config-if-range)#ip access-group 1 in
4)应用地址池
R1(config)#ip nat inside source list 1 pool A

二、PPP（Point-Point-Protocol，点对点协议）

2.1 基本概念

点到点协议（Point to Point Protocol，PPP）是为在同等单元之间传输数据包这样的简单链路设计的链路层协议。

  • PPP具有以下功能：
    1. PPP具有动态分配IP地址的能力，允许在连接时刻协商IP地址；
    2. PPP支持多种网络协议，比如TCP/IP、NetBEUI、NWLINK等；
    3. PPP具有错误检测能力，但不具备纠错能力，所以ppp是不可靠传输协议；
    4. 无重传的机制，网络开销小，速度快。
    5. PPP具有身份验证功能。
    6. PPP可以用于多种类型的物理介质上，包括串口线、电话线、移动电话和光纤（例如SDH），PPP也用于Internet接入。

2.2 认证方式

2.2.1 PAP认证

PAP（PasswordAuthenticationProtocol 密码认证协议）是 PPP协议集中的一种链路控制协议，通过两次握手建立认证，对等结点持续重复发送 ID/ 密码（明文）给验证者，直至认证得到响应或连接终止，常见于PPPOE拨号环境中。
首先被认证方向认证方发送认证请求（包含用户名和密码），以明文形式进行传输，认证方接到认证请求，再根据被认证方发送来的用户名去到自己的数据库认证用户名密码是否正确，如果密码正确，PAP认证通过，如果用户名密码错误，PAP认证未通过。

2.2.2 CHAP认证

CHAP (ChallengeHandshakeAuthenticationProtocol 质询握手认证协议)通过三次握手验证被认证方的身份（密文），在初始链路建立时完成，为了提高安全性，在链路建立之后周期性进行验证，目前在企业网的远程接入环境中用的比较常见。因此，CHAP比PAP的安全性要高很多。

2.3 PPP配置

2.3.1 PAP单向认证配置

R1(config)#username admin password ccie
R1(config)#interface s2/0
R1(config-if)#encapsulation ppp //封装为ppp协议
R1(config-if)#ppp authentication pap //ppp认证协议设置为pap

Router(config)#int s2/0
Router(config-if)#encapsulation ppp
Router(config-if)#ppp pap sent-username admin password ccie

2.3.2 CHAP双向认证配置

R2(config)#username R3 password ccie
R2(config)#int s3/0
R2(config-if)#encapsulation ppp
R2(config-if)#ppp authentication chap

R3(config)#username R2 password ccie
R3(config)#int s3/0
R3(config-if)#encapsulation ppp
R3(config-if)#ppp authentication chap
