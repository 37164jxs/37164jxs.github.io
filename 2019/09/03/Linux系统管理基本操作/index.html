Linux系统管理基本操作

Sep 03, 2019点击

一、基本概念

1.1 命令提示符

[root@localhost~]#
root:当前登录的用户名
localhost：本机的主机名
~:用户当前所在目录，~代表家目录
root 用户的家目录是：/root
普通用户的家目录是：/home
#:当前登录的用户为管理员用户
$：当前登录的用户为普通用

1.2 切换用户：su

1.3 显示当前目录所在对的完整路径：pwd

1.4 目录切换：cd

cd .. 返回上一级目录  
cd 返回家目录

二、常用的shell命令介绍

2.1 文件管理命令：ls–列表显示目录内容

2.1.1 颜色区分类别

蓝色代表目录（文件夹）
灰色代表普通文件
绿色代表可执行文件
红色代表压缩文件
浅绿色代表链接文件（快捷方式）

2.1.2 选项说明

-a 显示所有文件，包括隐藏文件（以.号开头的文件）。
-l 以长格式（内容更详细）显示文件或目录的详细信息。
-h 以 K、M、G 等单位显示文件大小（ 默认为字节），提高可读性。

2.1.3 通配符的使用

“?”可以匹配文件名中的一个任意字符
“*”可以匹配文件名中的任意多个字符。
举例：

显示/etc 目录下文件名以 ns 开头，扩展名是 conf 的文件的详细信息
ls –l /etc/ns*.conf
显示/etc 目录下文件名以 v 开头，文件名一共是 4 个字符的文件的详细信息
ls –l /etc/v???

2.2 创建目录：mkdir–要考虑 权限问题

  • -p 创建连续目录
    [root@localhost /]:#mkdir - p /media/cdrom
  • -m 创建目录同时设置权限
    [root@localhost /]:#mkdir - m 775 test

2.3 删除目录：rmdir

  • -p 删除连续目录

2.4 删除文件：rm–默认只删除文件，不删除目录

  • -f 无提示，直接删除

2.5 复制文件或目录：cp

复制文件
[root@localhost ~]# cp /bin/touch /root/test/toch1
[root@localhost ~]# cp /etc/inittab /etc/inittab.bat 将文件进行备份
复制目录
[root@localhost ~]# cp –R /tmp /root/mnt

2.6 移动/剪切文件或目录：mv

重命名：
[root@localhost ~]# mv /root/test/test1.txt /root/test/test2.txt

2.7 新建文件：touch

三、查看文件内容命令

3.1 显示文件内容：cat

-n 显示行号

3.2 分页显示文件内容：more/less

3.3 查看文件开头或末尾的部分：head/tail

默认显示10行内容
直接用-#（#代表任何数字）来改变显示行数

3.4 文件内容统计：wc

作用：用于统计指定文件中的行数、单词数、字节数。
语法：wc(选项)(参数)
常用选项：
-l 统计行数
-w 统计单词数
-c 统计字节数

四、查找命令

4.1 搜索文件与目录：find

作用：是在某个指定的路径下找我们需要的文件或目录，目标是文件或目录。 类似于在 Windows 系统中执行的“搜索”操作。
语法：find [查找范围] [查找条件表达式]
常用选项：
-name 按名称查找，允许使用通配符。
-type 按文件类型查找文件类型包括： 普通文件（f）、目录（d）、块设备 文件（b）、字符设备文件（c）
-user 按文件所有者查找，根据文件是否属于某个目标用户进行查找。
-size 按文件大小查找，使用“+”、“-”号设置超过或小于指定的大小 作为查找条件。常用的容量单位包括 k（注意是小写）、M、G

eg:
#find /etc -size +1024k -name "init*"
#find /etc -name 1.txt

4.2 查找文件内容：grep

作用：是在某个文件中找我们所需要的某部分内容，目标是字符串。则类似于在 WORD 中执行的“查找”操作。
语法：grep [选项] 查找条件 目标文件
常用选项：
-c ：计算找到 ‘搜寻字符串’ 的次数
-i ：忽略大小写的不同，所以大小写视为相同
-n ：顺便输出行号
-v ：反向选择，亦即显示出没有 ‘搜寻字符串’ 内容的那一行！
–color=auto ：可以将找到的关键词部分加上颜色的显示！

注意：grep 不支持通配符，而是使用正则表达式：
^ 表示以什么字符开头
$ 表示以什么字符结尾

  • “^word”表示以“word”开头
  • “word$”表示以“word”结尾
  • “^$”表示空行

五、其他辅助命令

5.1 查看历史命令：history

history [n] n 为数字，列出最近的 n 条命令
使用! 执行历史命令。
! number 执行第几条命令
! command从最近的命令查到以 command 开头的命令执行
!! 执行上一条

5.2 查看内部命令的帮助消息：help

5.3 查看命令的帮助手册：man

5.4 输入输出重定向：<和>/>>

5.5 管道符：|

六、文件权限

6.1 文件属性说明

  • 文件类型：
  • ：普通文件
    d：目录
    l：符号链接
    c：字符设备
    b：块设备
  • 文件权限：
    r：可读
    w：可写
    x：可执行
    -：没有权限

6.2 修改文件权限：chmod

6.2.1 字母方式修改

u：文件所有者 g：所属组 o：其他用户权限 a：所有用户
用+或者-来增加权限或减少权限

6.2.2 数字方式修改

r=4 w=2 x=1 -=0

6.3 修改文件的所有者和所属组：chown

举例：

chown teacher:teacher /test1 同时修改所有者与所属组
chown root /test1 修改所有者
chown :student /test1 修改所属组

6.4 特殊权限

6.4.1 ACL：

访问控制【针对其他用户当中的部份或个别用户设置特殊权限】

  1. setfacl -m u:stu1:rwx A：创建一个 ACL 规则，使 stu1 用户对 A 目录具有 rwx 权限
  2. getfacl A：查看 ACL 规则
  3. setfacl -x u:stu3 A：删除 stu3 对 A 目录的 ACL 权限，只删除了一条规则
  4. setfacl -b A：删除 A 目录的所有 ACL 规则
6.4.2 SBIT：粘滞位权限（强制位权限）

【针对其他用户，只能删除自己的文件，不能删除其他的用户的文件】
chmod o+t A

6.4.3 SGID

针对其他用户设置，以所属组的身份去执行。
chmod g+s A

6.4.4 SUID

以文件所有者的身份执行文件，只针对可执行文件。
chmod u+s A
